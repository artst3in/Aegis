# Four-Axis Evaluation — Shield Tier System (Redesigned)

## The Design

Pure linear progression on one axis: node count. 10 nodes. 6 tiers. DO is node 10, not a separate track. Diamond = all 10.

## 1. Mathematical Purity — ★★★★★ (5/5)

Strictly monotonic total order on one variable. Breakpoints at {0, 1, 5, 8, 9, 10}. Injective mapping: one count range → one tier. No ambiguity, no parallel axes, no edge cases. Previous design had two axes (node count + DO status) creating a Silver/Gold paradox. Eliminated.

## 2. Physical Purity — ★★★★★ (5/5)

Each tier = observable device state. None = no PIN prompt. Cyan = PIN exists. Bronze-Silver = feature count visible in skill tree. Gold = all hexes lit except DO. Diamond = all hexes including underground root. Every tier verifiable by looking at the security screen.

## 3. Emotional Purity — ★★★★★ (5/5)

Difficulty curve: Cyan is instant. Bronze takes 10 minutes. Silver takes a session. Gold requires finishing everything. Diamond requires a factory reset. Each step is harder than the last. The final jump is a different category entirely.

## 4. Social Purity — ★★★★★ (5/5)

Linear means no paradoxes. Higher tier ALWAYS means more security. "Why is yours higher?" has one answer: "More nodes lit." Status dashboard reads cleanly for everyone from grandmother to engineer.

## The Verdict

| Math | Physical | Emotional | Social | TOTAL |
|------|----------|-----------|--------|-------|
| 5 | 5 | 5 | 5 | **20/20** |

Structurally stronger than previous design. One axis instead of two eliminates the Silver/Gold paradox entirely.
